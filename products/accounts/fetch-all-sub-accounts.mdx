# `GET /v1/accounts`

#### Query parameters

<ParamField query="limit" type="integer">
  This endpoint is paginated. `limit` describes the size of the page you are querying.
</ParamField>

<ParamField query="cursor" type="string">
  The `cursor` is used to scroll to the next page. When making the first call to list all accounts, there is no need to pass in any cursor since the API has not returned any cursor back to you. Only use cursor when the API provides it. For more information, please see [Pagination](/getting-started/pagination)
</ParamField>

#### Response body

<ResponseField name="code" type="string" required>
  Response code
</ResponseField>

<ResponseField name="description" type="string" required>
  Response description
</ResponseField>

    <ResponseField name="results" type="object[]">
        <Expandable title="object"  defaultOpen="true">

                Creation timestamp.

            <ResponseField name="accountId" type="string" required>
                Account ID.
            </ResponseField>
            <ResponseField name="accountHolderId" type="string" required>

            </ResponseField>

                Account reference.

                Phone number.

                Bank Verification Number (BVN).
            </ResponseField>
            <ResponseField name="status" type="enum<string>" required>
                Account status.
            </ResponseField>

            </ResponseField>
            <ResponseField name="accountName" type="string" required>
                Account holder's name.

  </Expandable>

```bash Request
  --header 'Authorization: Bearer <token>' \
  --header 'accountId: <accountid>'
```
