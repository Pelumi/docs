## Overview
You may want to fetch transaction status or list of transactions for reconciliation purposes. Nomba provide different API for retriveing different typpes of transaction e.g debit/credit, virtual account, parent account transactions. Depending on what you need you can explore our fetch transaction API

<Note>
  The API for fetching related transaction associated with accounts is a paginated endpoint, we advice that you go through our pagination guide to understand how to call paginated endpoints or try live API interaction to see it in action.
</Note>

## Fetch Account Transactions

To Fetch account transaction within a specific timeframe make a GET 

<Info>
  The `cursor` is used to scroll to the next page. When making the first call to list all accounts, there is no need to pass in any cursor since the API has not returned any cursor back to you. Only use cursor when the API provides it. For more information, please see [Pagination](/getting-started/pagination)
</Info>

# `GET /v1/transactions/accounts`

<CodeGroup>
```bash Request
curl --request GET \
  --url https://api.nomba.com/v1/transactions/accounts \
  --header 'Authorization: Bearer <token>' \
  --header 'accountId: <accountId>'
```
