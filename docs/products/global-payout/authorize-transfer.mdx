# `POST /v1/global-payout/transfer/authorize`

AuthTransfer manages the complete transfer lifecycle with a single endpoint. It walks through every required step — from initiating a transfer to final authorization — ensuring nothing is missed before funds are moved.

<Tip>
  Before calling this endpoint, use [Fetch Exchange Rates](/docs/products/global-payout/fetch-exchange-rates) to get the latest rate for your currency pair. Pass the returned `exchangeRateId` as `lockedExchangeRateId` to guarantee that the rate shown to your customer is the rate used to fulfil the transaction.
</Tip>

## Payment methods

The `paymentMethod` field determines which additional fields are required. The table below summarises the key distinctions per rail:

| `paymentMethod` | Required fields (beyond the common set) | Notes |
|---|---|---|
| `BANK` (non-Canada) | `accountNumber`, `institutionName` | Destination bank account number; `institutionName` is the bank display name |
| `BANK` (Canada) | `accountNumber`, `institutionCode`, `beneficiary.beneficiaryEmail` | `institutionCode` is the 3-digit institution number |
| `MOBILE_MONEY` | `accountNumber`, `institutionName` | `accountNumber` is the recipient's phone number; use [List Mobile Money Providers](/docs/products/global-payout/mobile-money-providers) to get valid `institutionName` values |
| `INTERAC` | `beneficiary.beneficiaryEmail` | Canada only; `destinationCountryIsoCode` must be `CA` |
| `FASTER_PAYMENTS` | `accountNumber`, `institutionCode` | `institutionCode` is the 6-digit sort code (UK); `destinationCountryIsoCode` must be `GB` |
| `SEPA` | `accountNumber` (IBAN), `institutionCode` | `institutionCode` is the SWIFT/BIC code |

The common required fields for every payment method are: `amount`, `sourceCurrency`, `destinationCurrency`, `receiverName`, `sourceCountryIsoCode`, `destinationCountryIsoCode`, `authCode`, and `paymentMethod`.

#### Request body

<ParamField body="amount" type="number" required>
  The amount to transfer in the source currency.
</ParamField>

<ParamField body="sourceCurrency" type="string" required>
  The ISO 4217 currency code of the source account (e.g., `USD`).
</ParamField>

<ParamField body="destinationCurrency" type="string" required>
  The ISO 4217 currency code of the destination account (e.g., `EUR`).
</ParamField>

<ParamField body="receiverName" type="string" required>
  The full name of the recipient.
</ParamField>

<ParamField body="sourceCountryIsoCode" type="string" required>
  ISO 3166-1 alpha-2 country code of the sender (e.g., `NG`).
</ParamField>

<ParamField body="destinationCountryIsoCode" type="string" required>
  ISO 3166-1 alpha-2 country code of the recipient (e.g., `CD`). Must be `CA` for `BANK` (Canada) and `INTERAC`, `GB` for `FASTER_PAYMENTS`.
</ParamField>

<ParamField body="authCode" type="string" required>
  The authorization PIN/code to confirm the transaction.
</ParamField>

<ParamField body="paymentMethod" type="string" required>
  The payment rail to use. One of: `BANK`, `MOBILE_MONEY`, `INTERAC`, `FASTER_PAYMENTS`, `SEPA`.
</ParamField>

<ParamField body="accountNumber" type="string">
  The recipient's account number, IBAN, or phone number depending on the payment method.

  - `BANK`: destination bank account number
  - `MOBILE_MONEY`: recipient's phone number (e.g., `+243812345678`)
  - `FASTER_PAYMENTS`: destination account number (8 digits)
  - `SEPA`: recipient's IBAN
</ParamField>

<ParamField body="institutionCode" type="string">
  A routing code that identifies the recipient's financial institution. The format varies by payment method:

  - `BANK` (Canada): 3-digit institution number (e.g., `003`)
  - `FASTER_PAYMENTS`: 6-digit sort code (e.g., `204514`)
  - `SEPA`: SWIFT/BIC code (e.g., `DEUTDEDB`)
</ParamField>

<ParamField body="institutionName" type="string">
  The display name of the recipient's financial institution or mobile money provider.

  - `BANK` (non-Canada): bank name from the recipient's dropdown (e.g., `Rawbank`)
  - `MOBILE_MONEY`: mobile money provider display name (e.g., `Mpesa`, `Airtel Money`, `Orange Money`). Use the [List Mobile Money Providers](/docs/products/global-payout/mobile-money-providers) endpoint to retrieve valid values.
</ParamField>

<ParamField body="narration" type="string">
  A description or memo for the transfer.
</ParamField>

<ParamField body="lockedExchangeRateId" type="string">
  The `exchangeRateId` returned from a prior [Fetch Exchange Rates](/docs/products/global-payout/fetch-exchange-rates) call. When provided, the transfer is fulfilled at that exact rate, regardless of any subsequent rate movements.
</ParamField>

<ParamField body="callbackUrl" type="string">
  A URL to receive a webhook notification when the transaction status changes.
</ParamField>

<ParamField body="beneficiary" type="object">
  Beneficiary details. Required for `INTERAC` and `BANK` (Canada).
  <Expandable title="object">
    <ParamField body="beneficiaryEmail" type="string">
      The recipient's email address. Required for `INTERAC` and `BANK` (Canada).
    </ParamField>
    <ParamField body="securityQuestion" type="string">
      The security question for Interac e-Transfer or Canada bank transfer.
    </ParamField>
    <ParamField body="securityQuestionAnswer" type="string">
      The answer to the security question.
    </ParamField>
    <ParamField body="transitNumber" type="string">
      Bank transit number (Canada). 5-digit branch transit number.
    </ParamField>
    <ParamField body="beneficiaryAddress" type="string">
      Recipient's street address.
    </ParamField>
    <ParamField body="beneficiaryCity" type="string">
      Recipient's city.
    </ParamField>
    <ParamField body="beneficiaryState" type="string">
      Recipient's state or province.
    </ParamField>
    <ParamField body="beneficiaryPostCode" type="string">
      Recipient's postal or zip code.
    </ParamField>
    <ParamField body="phoneNumber" type="string">
      Recipient's phone number.
    </ParamField>
  </Expandable>
</ParamField>

#### Response body

<ResponseField name="wtTransactionId" type="string" required>
  The unique GlobalPayout transaction ID. Use this to track the transfer via [Fetch Transaction](/docs/products/global-payout/fetch-transaction).
</ResponseField>

<ResponseField name="coreTransactionId" type="string" required>
  The internal core transaction reference.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current processing status (e.g., `PROCESSING`, `COMPLETED`, `FAILED`).
</ResponseField>

<ResponseField name="coreStatus" type="string">
  The core system status (e.g., `PAYMENT_SUCCESSFUL`).
</ResponseField>

<ResponseField name="type" type="string">
  Transaction type. Will be `TRANSFER` for cross-border disbursements.
</ResponseField>

<ResponseField name="prettyStatus" type="string">
  A human-readable status label (e.g., `Successful`, `Processing`).
</ResponseField>

<ResponseField name="meta" type="object">
  Transaction metadata including currency details, amounts, and routing info. Contains only transaction-relevant fields — no vendor or internal system data.
</ResponseField>
