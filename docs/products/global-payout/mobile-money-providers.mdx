# `GET /v1/global-payout/mobile-money/providers`

Returns the available mobile money providers for your region. Use this endpoint to populate a provider selector in your UI before initiating a `MOBILE_MONEY` transfer.

The `code` from this response maps to `institutionCode` and the `displayName` maps to `institutionName` in the [Authorize Transfer](/docs/products/global-payout/authorize-transfer) request.

<CodeGroup>
```bash Request
curl --request GET \
  --url https://api.nomba.com/v1/global-payout/mobile-money/providers \
  --header 'Authorization: Bearer <token>' \
  --header 'accountId: <accountId>'
```

```json Response
{
  "code": "00",
  "description": "Success",
  "data": [
    { "code": "mpesa", "displayName": "Mpesa" },
    { "code": "airtel", "displayName": "Airtel Money" },
    { "code": "orange", "displayName": "Orange Money" }
  ]
}
```
</CodeGroup>

#### Response body

<ResponseField name="data" type="array" required>
  List of available mobile money providers.

  <Expandable title="provider object" defaultOpen="true">
    <ResponseField name="code" type="string">
      The provider code. Use this as `institutionCode` in the Authorize Transfer request.
    </ResponseField>
    <ResponseField name="displayName" type="string">
      Human-readable provider name. Use this as `institutionName` in the Authorize Transfer request.
    </ResponseField>
  </Expandable>
</ResponseField>
